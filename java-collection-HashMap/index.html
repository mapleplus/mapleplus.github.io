# HashMap

# HashMap 重要属性

  • Q：为什么容量必须是 2 的 n 次方？
  1. HashMap 为了提升性能，使用位运算来计算索引，如果容量是 2 的 n 次方，那么索引计算公式 (n-1) & hash，相当于取 hash 的低 n 位，这样计算效率更高
  2. 在计算索引的时候，根据 hashcode 高位是否为 1，和容量 & 运算时，如果原容量高位为 0，不发生移动，如果为 1 则移动到原索引 + 原容量的新位置，最好情况保证一半在数组前半部分，另一半在数组后半部分，数据更加均匀了！
