# 批次大小的计算 int batchSize = Math.max(1000, dataList.size() / (Runtime.getRuntime().availableProcessors() * 4));

首先，batchSize 的计算使用了 Math.max (1000, dataList.size () / (可用处理器数 *4))。这里的目标是确定每个批次的大小，以平衡任务分配的粒度。可用处理器数乘以 4 是为了确定一个合理的线程数，但为什么要乘以 4 呢？因为要创建足够多的任务以充分利用 CPU 资源，同时避免创建过多的小任务导致调度开销过大。当数据量较小时，批次大小被限制为至少 1000，防止批次过小，从而减少任务数量，避免过多的小任务影响性能。

# 分片处理逻辑

# 目的

将一个大列表 按指定大小 batchSize 分割成多个连续的子列表批次（Batch），返回这些批次组成的流，方便后续并行或分批处理。

  1. 计算 size，然后计算需要分多少部分，这里用到了 (size + batchSize -1) /batchSize，这是常见的计算分页或分块的方法，可以避免余数的问题。
  2. 通过 IntStream.range 生成从 0 到 parts-1 的整数，每个整数 i 对应一个子列表的起始和结束索引。
  3. 使用 subList 方法，并且结束索引是 Math.min (size, (i+1)*batchSize)，这样可以处理最后一个批次可能不足 batchSize 的情况。

示例：

  • → 批次范围：
    • → [0 , 3) 3 个元素
    • → [3 , 6) 3 个元素
    • → [6 , 9) 3 个元素
    • → [9, 10) 1 个元素

返回值类型 Stream<List<T>>

  • 特点
    • 每个元素是一个子列表 List<T>
    • 子列表是原始列表的视图（通过 subList 生成），修改子列表会影响原始列表。
    • 流式处理适合后续的并行操作或链式处理。
